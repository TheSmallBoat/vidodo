## 核心技术模块 Real vs Mock 评估

> 本节明确区分控制面（已真实实现）与执行面（当前为确定性模拟）。

### 控制面（已真实实现）

| 模块 | 评估 | 说明 |
|------|------|------|
| IR 数据模型 | **真实** | 90+ 公共类型，完整 serde round-trip |
| Validator | **真实** | 跨引用检查 show_id/section/layer/asset 一致性 |
| Compiler | **真实** | DSL → StructureIr + PerformanceIr + VisualIr + Timeline，真实 IR 变换 |
| Patch Manager | **真实** | 完整状态机：check → apply → rollback → deferred_rollback → checkpoint |
| Capability Layer | **真实** | 39 能力注册 + 路由 + 授权策略引擎 + MCP 映射 |
| Storage / Registry | **真实** | SQLite 持久化 + 文件 I/O + 资产 ingest |
| Trace Writer | **真实** | JSONL 文件写入 + 结构化 manifest + bar 过滤 |
| Evaluation | **真实** | 四维评分（sync / transition / resource_stability / patch_safety） |

### 执行面（当前为确定性模拟 / Stub）

| 模块 | 评估 | 文档规划的目标技术 | 当前状态 |
|------|------|-------------------|---------|
| Scheduler | **离线仿真** | 实时调度器（wall-clock tick loop） | 同步循环 `scheduler_time_ms += 1_000`，无真实计时 |
| Audio Backend | **Mock** | SuperCollider scsynth (OSC 桥接) | `FakeBackendClient` 返回硬编码 "ok" ack |
| Audio Export | **静默输出** | ffmpeg 混音导出链路 | 生成全零 PCM 的 RIFF WAV |
| Audio Analyzer | **Stub** | Python librosa/essentia | 返回硬编码 BPM/RMS/peak |
| Visual Backend | **Mock** | Rust + wgpu (GLSL 着色器输入) | `VisualReferenceBackend` 记日志，无 GPU |
| Visual Runtime | **事件日志** | wgpu 渲染循环 + scene graph | 读 trace events → 输出 scene acks JSON |
| Lighting Backend | **Mock** | DMX / ArtNet / sACN | `LightingReferenceBackend` 记日志，无硬件 |
| Lighting Runtime | **事件日志** | 灯光控制台驱动 | 读 trace events → 输出 cue acks JSON |
| GLSL Demo | **不存在** | shader 创作输入格式 | 无任何 .glsl / .wgsl / .frag / .vert 文件 |
| IPC / 进程间通讯 | **不存在** | Core → Audio/Visual 进程分离 | 全部 in-process 同步调用 |

### 差距分析

系统当前是一个**完整的控制面 + 确定性仿真的编排框架**。"大脑"（编译 → 调度 → 补丁 → 追踪 → 评估）真实可用；"肌肉"（音频输出、GPU 渲染、灯光控制）均为 mock。

优先补齐路径（按文档设计优先级）：

1. **SuperCollider 音频桥接** — 文档 13 号 Route 1（最高优先），需 OSC client crate + scsynth 进程管理
2. **wgpu 视觉渲染原型** — 文档 13 号 + 27 号，需 wgpu + winit + naga（GLSL→SPIR-V）
3. **GLSL Demo Shader** — 作为视觉引擎的首个可见交付物
4. **Python 分析管线** — librosa/essentia 替换 stub analyzer
5. **DMX / ArtNet 灯光输出** — 文档 29 号设计
