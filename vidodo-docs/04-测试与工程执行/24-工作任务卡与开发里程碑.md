# 工作任务卡、里程碑与路线图

日期：2026-04-01
状态：Draft
前置阅读：14-MVP技术架构图与Phase0实施清单.md，15-功能模块设计方案总览.md，22-编码前置完备性审计报告.md

## 1. 文档目的

本文将 14 号文档中的 Workstream A-F 拆解为可分配、可追踪的工作任务卡，并定义里程碑节点与依赖关系。

## 2. 任务卡规范

### 2.1 任务卡字段

| 字段 | 说明 |
| --- | --- |
| 卡号 | WS{工作流字母}-{序号}，例如 WSA-01 |
| 标题 | 动词开头的简短描述 |
| 工作流 | 所属 Workstream |
| 依赖 | 前置任务卡号 |
| 交付物 | 必须产出的工件 |
| 验收标准 | 可客观判断的完成条件 |
| 角色 | 推荐执行角色 |
| 规模 | S / M / L（1-2 / 3-5 / 5+ 工作日） |

### 2.2 任务状态

- `todo` → `in_progress` → `review` → `done`
- 允许 `blocked`（需记录阻塞原因与依赖卡号）

# Phase 1：能力层服务化与适配扩展

## 14. Phase 1 目标

Phase 0 证明了最小闭环（外部规划 → 编译 → 执行 → trace → patch → 评估）可工作。

Phase 1 的目标是：

1. 把 avctl 的业务语义提升为统一能力层（Capability Layer），使 CLI 和 MCP 共享同一能力面。
2. 用 HTTP 服务化包装能力层，使外部 Agent 可远程调用。
3. 实现 MCP 工具面，使 Agent 可发现、校验和提交结构化请求。
4. 扩展灯光运行时骨架，证明三通道（音频 + 视觉 + 灯光）统一调度可行。

Phase 1 明确不做的事：

- 不做真实硬件后端集成（Phase 2）
- 不做分布式多节点运行时
- 不做 GUI 产品
- 不做完整权限体系

# Phase 2：演出系统化

## 22. Phase 2 目标

Phase 1 证明了 CLI / HTTP / MCP 三入口语义等价和 audio / visual / lighting 三运行时统一调度。

Phase 2 的目标是：

1. 在 Core 中固定统一适配插件协议边界（Adapter Plugin Contract），使后续接入任意硬件后端只需实现一个 adapter plugin。
2. 固定外部资源 HUB 描述对象，使音频素材、GLSL 场景、灯光 preset 等资源以标准化引用方式被 Compiler / Runtime 消费。
3. 定义分布式装置部署层的正式对象（node / deployment / transport），为后续多节点编排提供稳定建模基础。
4. 引入运行时健康监控与降级策略骨架，为长时运行稳定性做基础建设。
5. 完善 revision 生命周期 HTTP handler（publish / archive），补全 Phase 1 遗留 stub。

Phase 2 明确不做的事：

- 不做真实硬件驱动实现（由独立 adapter 仓库承接）
- 不做消息总线完整实现（只定义 TransportContract 对象）
- 不做 GUI 产品
- 不做 MIDI / OSC 外部控制器实时桥接（Phase 3）
- 不做完整权限模型（仅声明授权占位）

# Phase 3：统一适配协议与后端操作化

## 31. Phase 3 目标

Phase 2 建立了适配插件注册表、资源 HUB 注册表、分布式部署对象模型与健康决策骨架。

Phase 3 的目标是：

1. 把 30 号文档 §4.2 定义的 7 方法统一插件协议形式化为 Rust trait，使三类后端共享同一协议边界。
2. 实现适配插件生命周期管理（加载、就绪检查、停机），脱离纯内存桩模式。
3. 为资源 HUB 增加 SQLite 持久化和资源解析缓存，使注册数据跨运行保持。
4. 实现最小授权策略引擎，使 capability 请求可被策略拒绝或降级。
5. 基于 trait 实现三类 Reference Backend Adapter（audio / visual / lighting），替换现有 FakeBackendClient。
6. 在 avctl / core-service / mcp-adapter 三入口完成新能力集成与端到端验证。

Phase 3 明确不做的事：

- 不做真实硬件驱动（ALSA、CoreAudio、DMX、Art-Net）绑定——仅参考实现
- 不做多节点分布式编排运行时（Doc 31 §7 结论：独立仓库）
- 不做 GUI 或 Web 前端
- 不做资源 HUB 市场 / CDN 分发
