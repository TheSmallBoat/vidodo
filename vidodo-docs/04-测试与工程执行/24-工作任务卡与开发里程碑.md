# 工作任务卡、里程碑与路线图

日期：2026-04-01
状态：Draft
前置阅读：14-MVP技术架构图与Phase0实施清单.md，15-功能模块设计方案总览.md，22-编码前置完备性审计报告.md

## 1. 文档目的

本文将 14 号文档中的 Workstream A-F 拆解为可分配、可追踪的工作任务卡，并定义里程碑节点与依赖关系。

## 2. 任务卡规范

### 2.1 任务卡字段

| 字段 | 说明 |
| --- | --- |
| 卡号 | WS{工作流字母}-{序号}，例如 WSA-01 |
| 标题 | 动词开头的简短描述 |
| 工作流 | 所属 Workstream |
| 依赖 | 前置任务卡号 |
| 交付物 | 必须产出的工件 |
| 验收标准 | 可客观判断的完成条件 |
| 角色 | 推荐执行角色 |
| 规模 | S / M / L（1-2 / 3-5 / 5+ 工作日） |

### 2.2 任务状态

- `todo` → `in_progress` → `review` → `done`
- 允许 `blocked`（需记录阻塞原因与依赖卡号）
